Jsx


O que é JSX

Um exemplo de código JSX é:

const element = h1 className = "title" hello World! h1

Não é apenas HTML nem JS

JSX serve para descrever como a nossa User Interface (UI) deve ser

JSX não é interpretado pelo browser, por este motivo devemos utilizar um transpiler (em geral babel) para realizar essa transpilação.

Ao contrário do passado, em vez de colocar JavaScript em HTML, o JSX permite-nos colocar HTML em JavaScript.

Este codigo Jsx que vimos, após o transpiler fica:

const element = React.createElement(

"h1",

{className: "title"},

"Hello World"

)


Com esse codigo preparamos o nosso elemento para ser exibido no DOM

Vantagens od JSX

Similaridade com o HTML

Melhor integração entre o HTML e o JavaScript

Quem utiliza JSX

Preact (parecido com react)

Inferno (parecido com react)

React-Native

Vuejs (Opcional)

Dentro dessa pasta criamos um arquivo chamado "hello-world.html" com a estrutura basica html e uma div com id="root"

Adicionamos o script do react: https://reactjs.org/docs/cdn-links.html

Dentro da pasta HelloWorld, criamos um arquivo js "hello-world.js"; Dentro desse arquivo chamamos o metodo "ReactDOM.render()"; Dentro desse método vamos ter 2 parametros

Segundo parâmetro: Dizer onde queremos "injetar" este elemento react que criamos e isso faz-se com o "document.getElementById("root")"

ReactDOM.render(

views


Como é que o react atualiza as views

No JavaScript nós usamos o DOM:

let myBtn = document.getElementById("btn")
myBtn.addEventListener("click", function(event) { return alert ("Botão clicado!")})

No react, usamos o Virtual DOM ou tambem chamado como React DOM

O virtual DOM é uma reprensentação em JavaScript do DOM original!

Quando as alterações/eventos acontecem, primeiro o react atualiza o virtual DOM, depois, ele analisa as mudanças que são precisas acontecer no DOM real e no fim é realizado a atualização conciliando o virtual DOM com o DOM original

Porquê que o REACT usa o virtual DOM e não o DOM diretamente?

Aceder ao DOM é mais lento do que as operações em JavaScript

O Virtual DOM automaticamente encontra a maneira mais economica de atualizar o DOM (atualizando unicamente os elementos necessários)

Entender os elementos

Os menores blocos do React são os elements, através deles é que descrevemos o que vai aparecer no nosso ecra

const element = h1 Hello, World h1

Diferente dos elementos do DOM, os elementos do React são objetos simples e "baratos" de criar

O virtual DOM é o responsavel por atualizar o DOM e exibir esses elementos no ecra

Para renderizar um elemento, primeiro escolhemos um nó do DOM, ex:

div id="root" div

Tudo dentro desse nó será atualizado pelo Virtual DOM

Agora para inserirmos o nosso elemento no ecra, podemos rodar:

const element = h1 Hello, World h1
ReactDOM.render(element, document.getElementById("root"))


Create React App: Nosso primeiro Projeto


Para usarmos o create react app, precisamos de ter instalado o node e versão do npm > 5.2

Para ver se está tudo instalado, no terminal escrevemos: node -v; npm -v

Agrupar Components


Vamos criar uma nova pasta "Agrupar Components" e iniciar la o react. Depois disso, na pasta "src" vamos criar a pasta "Components". Dentro dessa pasta "Components" criamos outra com o nome "planets".

Dentro da pasta "planets" criamos um arquivo chamado "index.js". Dentro desse ficheiro criamos a sua função com o retorno de uma div com um h3 "Planet List"

No App.js, importamos Planets from Components/planets

No return da função App, retornamos uma div com o componente < Planets />

No index.js, onde temos a função do planets, vamos substituir a div do return, por "fragments" <>

Vamos criar outra pasta, dentro da pasta "Planets" com o nome "planet" com um arquivo index.js

Nesse arquivo vamos criar um novo componente (função) que retorne uma div com um h4 "Marte", um p "descrição...lorem ipsum..." e uma imagem desse planeta (podemos ir ao wikipedia)

Basicamente criamos um componente que usa outro componente

Podemos repetir o component < Planet /> 2x, para ver que conseguimos repetir as vezes que quisermos! (ver o resultado no browser)
