views


Como é que o react atualiza as views

No JavaScript nós usamos o DOM:

let myBtn = document.getElementById("btn")
myBtn.addEventListener("click", function(event) { return alert ("Botão clicado!")})

No react, usamos o Virtual DOM ou tambem chamado como React DOM

O virtual DOM é uma reprensentação em JavaScript do DOM original!

Quando as alterações/eventos acontecem, primeiro o react atualiza o virtual DOM, depois, ele analisa as mudanças que são precisas acontecer no DOM real e no fim é realizado a atualização conciliando o virtual DOM com o DOM original

Porquê que o REACT usa o virtual DOM e não o DOM diretamente?

Aceder ao DOM é mais lento do que as operações em JavaScript

O Virtual DOM automaticamente encontra a maneira mais economica de atualizar o DOM (atualizando unicamente os elementos necessários)

Entender os elementos

Os menores blocos do React são os elements, através deles é que descrevemos o que vai aparecer no nosso ecra

const element = h1 Hello, World h1

Diferente dos elementos do DOM, os elementos do React são objetos simples e "baratos" de criar

O virtual DOM é o responsavel por atualizar o DOM e exibir esses elementos no ecra

Para renderizar um elemento, primeiro escolhemos um nó do DOM, ex:

div id="root" div

Tudo dentro desse nó será atualizado pelo Virtual DOM

Agora para inserirmos o nosso elemento no ecra, podemos rodar:

const element = h1 Hello, World h1
ReactDOM.render(element, document.getElementById("root"))

Conhecer as Props


Função pura:

A função pura recebe duas propriedades: a b

A função pura não altera os valores das entradas

Sempre retorna o mesmo resultado para os mesmos tipos de entrada


function sum(a,b) {

return a + b

}


Função impura:

A função pura altera os valores das entradas


function withdraw(account,amount) {

account.total -= amount

}

O que são estados (STATES)


State é um objeto javascript que guarda os dados de um component e determina o seu comportamento

Quando um estado é atualizado o component é re-renderizado


Diferença entre props e state:

As props são passadas por outros components (component pai passa as suas props para um component filho)

O state pertence a cada component e tambem quando as props são atualizadas, o component vai re-renderizar novamente e o estado (state) quando atualizado também vai re-renderizar


Component de class vs Component functional:

Antes so era permitido usar state em components de class, por isso é que esses components eram tão mais importantes

Agora, podemos usar states em components functional, graças aos hooks


declarar um state num component de classe

constructor(props){

super(props)

this.state = {

count: 0

}

}


Para manipular um component num component de classe, usamos um metodo chamado "setState()"

incrementCount(){

this.setState((state) => {

return {count: state.count + 1}

})

}


declarar um state num component functional

import React, { useState } from "react"

const [count, setCount] = useState(0)


Para manipular um component num component

setCount(count + 1)
